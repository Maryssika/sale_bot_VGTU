# Техническое задание (ТЗ)

## Название проекта
Telegram-бот для поиска товаров и прогноза цен на Wildberries с интеграцией Google Shopping

---

## 1. Общие сведения
Создать Telegram-бота, который позволяет пользователям:

- Искать товары по ключевым запросам на маркетплейсе Wildberries
- Получать кэшированные результаты поиска
- Просматривать статистику кэша
- Получать прогноз цен на 7 дней вперёд
- Искать товары в Google Shopping с парсингом результатов
- Вести логирование и собирать метрики

---

## 2. Функциональные требования

### 2.1 Команды бота

| Команда      | Описание                                    | Пример использования  |
|--------------|---------------------------------------------|-----------------------|
| `/start`     | Инструкция по использованию бота            | `/start`              |
| `/wb`        | Поиск товаров по запросу на Wildberries     | `/wb смартфон`        |
| `/cacheinfo` | Информация о кэше по запросу                | `/cacheinfo смартфон` |
| `/forecast`  | Прогноз цен на 7 дней для запроса           | `/forecast смартфон`  |
| `/google`    | Поиск товаров в Google Shopping             | `/google смартфон`    |
| `/search`    | Поиск товаров по запросу на Wildberries и Google Shopping| `/search смартфон`    |

---
  
- ### 2.1 Поиск товаров (`/wb`)

- При получении команды бот проверяет наличие кэшированных данных по запросу.
- Если данные свежие (меньше 2 часов) — возвращает кэш.
- Если нет — обращается к API Wildberries и сохраняет результаты в MongoDB.
- Отправляет пользователю до 5 товаров с:
  - Названием
  - Ценой
  - Брендом
  - Рейтингом
  - Ссылкой на товар

---

### 2.2 Поиск в Google Shopping (`/google`)

- Для парсинга результатов используется **Selenium WebDriver** с автоматическим управлением драйвером через **WebDriverManager**.
- При вводе команды `/google <запрос>` выполняется поиск на Google Shopping.
- Извлекается до **10 товаров**, включая:
    - Название товара
    - Цену
- Формируется и отправляется пользователю **текстовый ответ** с результатами.
- Обрабатываются возможные ошибки:
    - отсутствие результатов
    - изменение структуры страницы
    - блокировка парсера
- Ведётся логирование всех запросов и ошибок для анализа и отладки.
- Реализованы меры по снижению риска блокировки:
    - установка корректного User-Agent
    - использование задержек и ограничений скорости запросов

---

### 2.3 Поиск товаров (`/search`)

- При получении команды бот проверяет наличие кэшированных данных по запросу.
- Если данные свежие (меньше 2 часов) — возвращает кэш.
- Если нет — обращается к API Wildberries и Google Shopping и сохраняет результаты в MongoDB.
- Отправляет пользователю до 5 товаров с:
  - Названием
  - Ценой
  - Брендом
  - Рейтингом
  - Ссылкой на товар

---

### 2.4 Кэширование и статистика (`/cacheinfo`)

- Возвращает:
    - Количество товаров по запросу
    - Время последнего обновления кэша
    - Количество поисков этого запроса

---

### 2.5 Прогноз цен (`/forecast`)

- Генерирует прогноз цен на 7 дней на основе истории цен.
- Формат отображения:

>    📈 Прогноз цен для запроса: смартфон
>
>    📅 10.06.2025 → 15000 ₽
>
>    📅 11.06.2025 → 14950 ₽

---

## 3. Нефункциональные требования

- Использовать MongoDB для хранения данных и логов.
- Логировать все запросы и ошибки.
- Обеспечить стабильность и отказоустойчивость.
- Хранить конфиденциальные данные (API-ключи, токены) в защищённой конфигурации.
- Использовать TLS для всех внешних соединений.
- Обработка ошибок с информативными логами.

---

## 4. Архитектура

- Компоненты:
  - **TelegramBot** — обработка команд и взаимодействие с пользователем
  - **WildberriesApiClient** — получение данных с Wildberries
  - **GoogleShoppingParser** — парсинг Google Shopping с помощью Selenium
  - **MongoDBService** — работа с базой данных
  - **ForecastService** — генерация прогноза цен

---

## 5. Технологии

- Язык: Java 17+ 
- Библиотеки:
  - `org.telegram.telegrambots` — для Telegram API
  - MongoDB Java Driver
  - JSON-парсеры (Jackson или org.json)
  - Selenium WebDriver с WebDriverManager
